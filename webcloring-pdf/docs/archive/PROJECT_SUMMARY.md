# INTEROJO 포털 자동화 시스템 개선 프로젝트 요약

## 1. 프로젝트 개요
본 프로젝트는 INTEROJO 포털에서 자재 요청 문서를 자동으로 검색하고, PDF로 저장하며, Excel 파일로 데이터를 정리하는 Python 자동화 시스템입니다. 사용자 친화적인 GUI 인터페이스와 자동화 모드를 지원합니다.

## 2. 개선 목표
사용자의 요청에 따라 다음과 같은 목표를 가지고 프로젝트 개선 작업을 수행했습니다.
*   **코드베이스 정리**: 불필요한 파일 제거 및 논리적인 구조 재편.
*   **자동화 견고성 향상**: UI 자동화의 안정성과 신뢰도를 높여 오류 발생률 감소.
*   **기능 확장**: 사용자 편의성을 위한 새로운 기능 추가.
*   **설정 관리 효율화**: GUI를 통한 설정 변경 및 저장 기능 구현.

## 3. 수행된 작업 및 개선 사항

### 3.1. 프로젝트 구조 정리
*   **불필요한 파일 및 폴더 삭제**: `build/`, `dist/`, `automation_package/`, `__pycache__/`, `automation.spec`, 중복된 `Material_Release_Request.xlsx` 등 빌드 아티팩트 및 임시/중복 파일을 제거하여 프로젝트를 경량화했습니다.
*   **논리적 폴더 구조 재편**:
    *   모든 파이썬 소스 코드를 `src/` 디렉토리(`core/`, `utils/`, `gui/`, `services/`, `config/`) 안으로 이동시켰습니다.
    *   자동화 결과물인 PDF 및 Excel 파일을 저장하는 `data/` 디렉토리를 새로 생성하여 `PDF/`와 `excel/`을 그 하위로 이동시켰습니다.
*   **내부 경로 수정**: `main.py`의 `sys.path`에 `src/`를 추가하고, `src/config/settings.py`에서 `base_dir` 및 `output` 관련 경로를 새 구조에 맞춰 수정하여 올바른 모듈 임포트 및 파일 접근을 보장했습니다.

### 3.2. UI 자동화 견고성 향상
기존 `time.sleep()` 기반의 고정 지연 시간을 제거하고, 동적인 `WebDriverWait` 조건을 사용하여 자동화의 안정성과 신뢰도를 대폭 향상시켰습니다.
*   **창 포커스 및 가시성 확보**: `setup_driver` 메소드에 `self.driver.maximize_window()`를 추가하여 브라우저 창이 항상 최대화되고 포커스를 유지하도록 했습니다.
*   **지능적인 대기 조건 적용**:
    *   `login_to_portal`: 로그인 후 메인 페이지 URL 변경 또는 특정 요소(`gnbWrap`) 출현까지 대기하도록 수정했습니다.
    *   `navigate_to_electronic_approval`: 전자결재 페이지 URL 변경 및 핵심 요소(`결재대기`, `결재진행`) 출현까지 대기하도록 수정했습니다.
    *   `navigate_to_completed_documents`: 완료문서함 페이지 이동 후 검색 폼(`searchFormName`) 출현까지 대기하도록 수정했습니다.
    *   `search_documents`: 검색 실행 후 결과 테이블(`//tbody//tr`)이 로드될 때까지 대기하도록 수정했습니다.
    *   `change_page_size`: 페이지당 글 수 변경 후 문서 테이블(`//tbody//tr[td]`)이 다시 로드될 때까지 대기하도록 수정했습니다.
    *   `process_document`: 문서 상세 페이지(`approval_view_wrap`) 로딩 및 뒤로 가기 후 문서 목록(`//tbody//tr[td]`) 복귀까지 대기하도록 수정했습니다.
    *   `process_document_list`: `collect_document_list` 내 iframe 처리 로직을 명확히 하고, 문서 목록 수집 시 빈 결과(`검색 결과가 존재하지 않습니다.`)를 명시적으로 처리하도록 로직을 추가했습니다.
*   **오류 처리 로직 개선**: `process_document_list` 내에서 개별 문서 처리 실패 시, 단순히 오류를 기록하고 스크린샷을 저장한 뒤 '뒤로 가기'를 통해 문서 목록으로 복귀하여 다음 문서 처리를 이어가도록 복원력을 강화했습니다.

### 3.3. 새로운 기능 구현

*   **헤드리스(Headless) 모드 지원**:
    *   `.env` 파일의 `HEADLESS_MODE` 설정을 통해 브라우저 창을 띄우지 않고 백그라운드에서 자동화를 실행할 수 있는 기능을 추가했습니다.
    *   `src/config/settings.py`에 `headless_mode` 속성을 추가하고 `src/core/portal_automation.py`의 `setup_driver` 메소드에서 이를 활용하도록 구현했습니다.
*   **GUI를 통한 설정 관리**:
    *   `src/gui/settings_dialog.py`를 통해 `HEADLESS_MODE`를 포함한 모든 주요 `.env` 설정(사용자 인증 정보, 검색 시작 날짜, 동적 필터링, 스케줄 등)을 GUI에서 직접 수정하고 저장할 수 있도록 기능을 개선했습니다.
    *   `src/config/settings.py`에 중앙화된 `save_env_settings` 메소드를 구현하여 `.env` 파일 저장 로직을 효율화했습니다.

## 4. 해결된 주요 문제점

*   **초기 코드 혼란**: 프로젝트 초기 상태의 정리되지 않은 파일 및 폴더 구조를 해결했습니다.
*   **설정 파일 접근 문제**: `.env` 파일과 설정 로딩 메커니즘을 명확히 하고, GUI를 통한 접근성을 높였습니다.
*   **UI 자동화 취약성**: `time.sleep()`에 의존하던 불안정한 대기 로직을 `WebDriverWait`으로 대체하여 UI 변경이나 네트워크 지연에 강한 자동화를 구축했습니다.
*   **Selenium `AttributeError` 및 `SyntaxError`**: 리팩토링 과정에서 발생했던 메소드 누락 및 구문 오류를 정확히 수정하여 코드의 안정성을 회복했습니다.
*   **XPath 오류**: 문서 목록 스크래핑 시 발생했던 `NoSuchElementException` 문제를 `starts-with(@href, 'javascript:getApprDetail')`를 활용한 더 견고한 XPath로 수정했습니다.
*   **오류 상황 처리**: 검색 결과가 없는 경우를 정상적인 시나리오로 인식하고, 오류 발생 시 자동화가 멈추지 않고 계속 진행되도록 처리 로직을 개선했습니다.

## 5. 다음 단계 (향후 계획)

현재 프로젝트는 매우 안정적이며 효율적으로 작동합니다. 향후 다음과 같은 추가 개선을 고려해볼 수 있습니다.

*   **로그 가독성 향상**: 터미널 출력 시 한글 깨짐 문제를 근본적으로 해결하기 위한 추가적인 인코딩 관련 조사를 진행합니다.
*   **상세 로깅 및 모니터링**: 자동화 실행에 대한 더 상세한 로그와 함께, 에러 발생 시 알림(이메일, 메신저 등) 기능을 구현하여 관리 효율성을 높입니다.
*   **PDF 저장 시 자동화 재시도**: PDF 저장 실패 시 자동으로 재시도하거나, 특정 PDF만 건너뛰고 다음 문서 처리를 진행하는 로직을 추가합니다.
*   **GUI 기능 확장**: 스케줄 관리, 현재 상태 모니터링 등 GUI의 기능을 더욱 풍부하게 만들어 사용자 편의성을 극대화합니다.
